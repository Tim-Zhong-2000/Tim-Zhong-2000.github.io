优势：更小的查找次数，在比较操作很费资源的情况有明显优势

2020.10.13

快速排序

数据结构：堆
若编号从0开始。编号为k的节点，左右两个孩子的编号分别是2k+1和2k+2。
若编号从1开始。两个孩子分别是2k和2k+1

大(小)顶堆：父节点值比子节点大(小)

内存中的堆与数据结构中的堆没有半毛钱关系

算法：堆排序
原理与选择排序相似，思路都是：

  1. 选择最大值
  2. 移出当前最大值
  3. 寻找下一个最大值…

堆排序有几个典型操作：

  1. 建堆(初始化)复杂度O(nlog(n))
    1. 父节点与两个子节点，三数比较，选出最大数放入父节点。
    2. 进入子节点重复操作

  1. 选择最大值
    对于数组：
    取出最后一位 temp
    把arr[0]移动到最后一位

  2. 重建
    此时arr[0]没有元素

    1. 对比0的子节点1，2，游离节点（假设1大），选出最大的节点1放入0
    2. 对比1的子节点3，4，游离节点（假设3大），选出最大的节点3放入1
    3. 反复操作
      rebuild(current, start, end){
    int leaf = 2*start+1;
    while(leaf <= end){
        // 比较相邻的节点,选中大的那一个
        if(leaf&lt;end && entry[leaf]&lt;entry[leaf+1]){
            leaf++;
        }
        if(entry[leaf]&lt;current){
            //current最大所以放在根节点
            entry[start]=current;
            break;
        }
        // 把大的一个节点移动到根节点
        entry[start] = entry[leaf];
        // 进入以leaf为根的更小的树
        start = leaf;
        leaf = leaf*2 + 1;
    }
}

堆排序中选择最大值操作简单，直接从堆顶取元素可以得到最大值，复杂度为O(1)
移出堆顶后需要维护自身，使得第二大的数置于堆顶，复杂度为O(log(n))…

算法效率分析：不考虑建堆时间，对于n个元素，需要做n次选择，每次维护复杂度O(log(n))，所以恒为O(nlog(n))。

2020-10-15

主题：突破nlog(n)的排序算法

计数排序 (counting sort)

假设n个元素都是正整数，最大值为m

把元素插入对应下标位置，只需要O(n)，需要m空间
合并成连续数组时需要扫描m长度的数组，复杂度为O(m)

排序复杂度为O(m+n)
优势：当n很大m很小时，很有优势
劣势：如果数据非常离散m大n小，不如快排等O(nlog(n))排序算法

radix sort

假设最大的数有k位，(10001有5位)
先对个位进行counting sort，得到第一轮结果
再对十位进行counting sort，得到第二轮结果
…
得到最终结果

复杂度O(kn)

2020-10-20

散列表(哈希表)

问题：如何存储100个手机号码（13511111111，13512341234……）对应的余额等信息，提供比较高效的查找速度。

散列表思路：将100个手机号码映射到100个不同的下标。假设这个从手机号到下标的转换函数为hashFunction。

| —- | —- | —- | —- |
| 手机号码 | 13511111111 | 13512341234 | …… |
| 数组下标 | 0 | 1 | …… |
| 余额 | 100 | 200 | …… |

可能出现的问题：哈希函数可能发生冲突，比如号码13511111111与号码1361111111经过哈希函数计算，结果都为0，都指向数组0号元素。因为哈希函数原理上会减少原来的信息量，所以哈希函数的冲突不可避免。但是改进哈希函数可以降低冲突发生的概率。

既然存在冲突，可以考虑让一个位置存放多个元素。先用哈希表确定下标，再从这个下标的有限个元素中搜索出要找的元素。
